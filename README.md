# 流

一个日程管理应用。

## 灵感

每个人都有属于自己的日常管理方案，因此对于用户而言，使用哪一款日程管理应用都存在学习成本。本应用基于**我们认为的**更加便捷的管理方案开发，**或许**比较适合大学生、公司等个人日程管理环境。

## 方案

我们将生活中的所有事情定义成**日程**与**事项**。所有日程和事项都可以标注其时间、地点、参加人列表、辅助性文字。所有日程和事项都可以被分组管理。

- **日程**是有一定持续时间的事情，例如课程、考试、会议。有开始时间与结束时间，它不需要被完成。

- **事项**分为两种：**任务**与**事件**，分别强调其是否需要被完成。

  - **任务**是需要在截止时间完成的事项，例如作业提交、表格上传等。任务未在规定时间完成会被置顶提示直到完成。

  - **事件**是标记某些未来有意义事项的发生时间，例如火车票抢票、生日等。事件不需要被完成，到达发生时间后也不会被特殊强调。

## 展示

计划采用多种展示方法。

- **时间线**：所有事情纵向按开始时间排列。**日程**使用较宽的方框表示，**事项**使用较窄的方框表示。所有
事情都不重叠。

- **日历**：日历模式中只提供**周视图**与**月视图**两种模式，
  - **周视图**：仅在屏幕上显示一周的信息，**日程**按照实际发生时间设置长度与位置。**事项**以窄方框表示，放置于发生时间处。**事项**可以覆盖于**日程**上；日程间不覆盖，左右并排放置；事项间不覆盖，以重要度上下排列放置。
  - **月视图**：仅标记当天是否有事情发生，如果有则按开始时间顺序将**日程**和**事项**列在下方。
